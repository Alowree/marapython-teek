::: details The "Double Backtick" Strategy

Using `<code>` tags inside Markdown feels like wearing a tuxedo to a backyard BBQ—it’s technically correct, but way more effort than it should be.

The "secret" to escaping backticks inside code spans in Markdown is actually built into the syntax: **use multiple backticks as delimiters.**

To display a backtick inside a code span, wrap the content in **double backticks** and add a single space at the start and end of the content. Markdown will treat those exterior spaces as padding and won't render them, but it allows the inner single backtick to be treated as literal text.

Here is your list cleaned up using pure Markdown:

- `:marks` - list of marks
- `ma` - set current position for mark A
- `` `a `` - jump to position of mark A
- ``y`a`` - yank text to position of mark A (Neovim + Prettier appears to remove the space for this one)
- `` `0 `` - go to the position where Vim was previously exited
- `` `" `` - go to the position when last editing this file
- `` `. `` - go to the position of the last change in this file
- ` `` ` - go to the position before the last jump (Neovim + Treesitter does not recognize the syntax)
- `:delmarks {marks}` - delete marks
- `:delmarks!` - get rid of all local marks
- `:ju[mps]` - list of jumps
- `Ctrl` + `i` - go to newer position in jump list
- `Ctrl` + `o` - go to older position in jump list
- `:changes` - list of changes
- `g,` - go to newer position in change list
- `g;` - go to older position in change list
- `Ctrl` + `]` - jump to the tag under cursor

Why this works

Markdown looks for a matching number of opening and closing backticks. By using two (or three, or four), you define a boundary that "ignores" any smaller number of backticks inside it.

| Requirement           | Markdown Syntax  | Result   |
| --------------------- | ---------------- | -------- |
| **Normal code**       | `` `code` ``     | `code`   |
| **Contains backtick** | ``` `` `a `` ``` | `` `a `` |
| **Double backticks**  | ``` ` `` ` ```   | ` `` `   |

**Pro Tip:** If you ever need to display ` ```` ` (double backticks) inside a code span, you would simply wrap them in _triple_ backticks and use the same spacing trick.

:::

## 脚注

Another point to note.[^longnote]

[^1]: This is the content of the first footnote.

[^2]: This is the content of the second footnote.

[^longnote]: This footnote might be longer and span multiple paragraphs. Indent subsequent paragraphs (usually by four spaces or one tab) to include them within the same footnote.
